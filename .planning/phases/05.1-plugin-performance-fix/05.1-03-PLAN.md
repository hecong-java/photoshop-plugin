    **Part C: Optimize fs.listDownloads (lines 608-629)**

    Current code reads entire file to get size:
    ```javascript
    'fs.listDownloads': async () => {
      const downloadsFolder = await ensureDownloadsFolder();
      const entries = await downloadsFolder.getEntries();
      const files = entries.filter((entry) => entry.isFile);
      const list = [];
      for (const file of files) {
        let size = 0;
        try {
          const content = await file.read({ format: formats.binary });
          size = content?.byteLength || 0;
        } catch {
          size = 0;
        }
        list.push({
          filename: file.name,

          size,
          modifiedTime: Date.now()
        });
      }
      return list;
    },
    ```

    Replace with optimized version that avoids reading file content:
    ```javascript
    'fs.listDownloads': async () => {
      const downloadsFolder = await ensureDownloadsFolder();
      const entries = await downloadsFolder.getEntries();
      const files = entries.filter((entry) => entry.isFile);
      const list = [];
      for (const file of files) {
        // Use nativePath for file info -- avoid reading entire file content
        // UXP entry does not expose size directly, but we can use 0 as placeholder
        // since the webapp primarily needs filename and path
        list.push({
          filename: file.name,
          path: file.nativePath || `${downloadsFolder.nativePath}/${file.name}`,
          size: 0,
          modifiedTime: Date.now()
        });
      }
      return list;
    },
    ```

    NOTE: UXP storage entries do not expose a `size` property directly. Setting `size: 0` is acceptable because the webapp primarily needs filenames and paths for the download list display.
