# Phase 05.1 Plan 01: Keyboard Shortcut Passthrough Summary

Whitelist-based keyboard shortcut passthrough hook forwarding PS shortcuts from webview to Photoshop via Bridge batchPlay actions.

## Implementation Details

### useKeyboardPassthrough Hook
- React hook (`useEffect` with empty deps) that attaches a capture-phase `keydown` listener
- Only activates inside UXP WebView environment (`isUXPWebView()` guard)
- Skips events when target is `INPUT`, `TEXTAREA`, `SELECT`, or `isContentEditable`
- Builds shortcut key string from modifier keys (`Ctrl`/`Shift`/`Alt`) + `event.key`
- Forwards only whitelisted shortcuts via `sendBridgeMessage('ps.executeShortcut', {...})`
- 16 supported shortcuts: Delete, Backspace, Ctrl+Z, Ctrl+Shift+Z, Ctrl+S, Ctrl+C, Ctrl+V, Ctrl+A, Ctrl+D, Ctrl+T, ArrowUp/Down/Left/Right, BracketLeft/Right

### ps.executeShortcut Bridge Handler
- Receives `{ key, ctrl, shift, alt }` payload from webapp
- Reconstructs shortcut key string and looks up batchPlay action descriptor
- Executes via `core.executeAsModal` + `action.batchPlay` for PS native operations
- Graceful error handling: returns `{ executed: false, reason }` instead of throwing

### Draw.tsx Integration
- Imports and calls `useKeyboardPassthrough()` in the Draw component
- Placed before the first `useEffect` block
