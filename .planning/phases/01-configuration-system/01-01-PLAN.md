  <name>Task 1: Add fs.readPluginConfig Bridge handler</name>
  <files>PS-plugin/ningleai/main.js</files>
  <action>
    Add a new handler `'fs.readPluginConfig'` to the `handlers` object in main.js.

    Implementation requirements:
    1. Accept payload with optional `filename` parameter (default: 'node-config.json')
    2. Use `localFileSystem.getEntryWithUrl('plugin:/' + filename)` to access plugin folder
    3. Read file content and parse as JSON
    4. Return `{ exists: true, data: parsedConfig }` on success
    5. Return `{ exists: false, data: null }` if file doesn't exist (not an error)
    6. Return `{ exists: false, data: null, error: errorMessage }` on JSON parse error
    7. Wrap in try-catch to handle all error cases gracefully

    Add this handler after the existing `'fs.deleteDownload'` handler, before `'fs.openDirectory'`.

    Pattern reference (from existing code):
    ```javascript
    'fs.readPluginConfig': async (payload) => {
      const { filename = 'node-config.json' } = payload || {};
      try {
        const configFile = await localFileSystem.getEntryWithUrl(`plugin:/${filename}`);
        if (!configFile || !configFile.isFile) {
          return { exists: false, data: null };
        }
        const content = await configFile.read();
        const config = JSON.parse(content);
        return { exists: true, data: config };
      } catch (error) {
        return {
          exists: false,
          data: null,
          error: error.message || 'Failed to read config'
        };
      }
    },
    ```

    Note: This is a JavaScript Bridge handler in the UXP runtime, not testable with Vitest. Verification is via code inspection and manual UXP testing (covered in Plan 04).
