**1. `lemongrid.fetch` handler** (per D-98):
- Same pattern as `comfyui.fetch` handler (lines 791-875)

- Extract `accessToken` from `lgSettings` and inject as `Authorization: Bearer ${token}` header
- The handler reads `lemongrid` key from settingsStorage (set by webview via `settings.set` Bridge call during login)
- Same response handling: JSON response, binary response (image/octet-stream -> base64 `{ __base64__: true, data, contentType, dataUrl, byteLength }`), text fallback

- Same error format: `{ code, message, url }`
- NO retryOnAbort parameter needed (LemonGrid is not localhost)

**2. `lemongrid.websocket` handler** (per D-23, D-24):
- New handler with no exact analog in main.js
- Accepts `{ taskId }` payload
- Read `lgSettings` from settingsStorage to get `serverUrl` and `accessToken`
- Build WS URL: `lgSettings.serverUrl.replace(/^http/i, 'ws') + '/ws/v1/realtime?token=' + lgSettings.accessToken`
- Create WebSocket in main.js: `const ws = new WebSocket(wsUrl)`
- On `ws.onmessage`: relay to webview via `webviewEl.postMessage({ type: 'lemongrid.ws.message', taskId, data: JSON.parse(wsMessageData) })`
- On `ws.onerror` / `ws.onclose`: relay to webview via `webviewEl.postMessage({ type: 'lemongrid.ws.close', taskId, code, reason })`
- Return a `{ connectionId: string }` so webview can reference this connection for cleanup
- Store active WS connections in a Map: `const lgWsConnections = new Map()` at module level. Key = connectionId. On cleanup, call `ws.close()` and delete from map.
- Add a companion handler `lemongrid.websocket.close` that accepts `{ connectionId }` and closes the WS

**3. `lemongrid.uploadAsset` handler** (per D-100):
- Same pattern as `comfyui.uploadImage` handler (lines 878-946)
- Accepts `{ url, filename, base64Data, mimeType, libraryType }` payload

- Build multipart with `name="file"` (LemonGrid's field name, NOT `name="image"`)

- Same base64-to-binary conversion, same response handling
- URL is provided by the webview (full LemonGrid upload URL)

**1. Create lemongrid-auth.ts** (per D-72, D-79, D-86, D-42):
- New file: `code/webapp/src/services/lemongrid-auth.ts`
- `lemongridFetch(url, options?, timeout?)` -- mirrors bridgeFetch pattern but calls `sendBridgeMessage('lemongrid.fetch', ...)` instead of `'comfyui.fetch'`. Shapes response into same Response-like object with `ok`, `status`, `json()`, `blob()`, `arrayBuffer()` methods. In browser mode (non-UXP), reads token from lemongridStore and adds Authorization header manually.
- `loginToLemonGrid(serverUrl, username, password)` -- POST to `{serverUrl}/api/v1/auth/login` with `{ username, password }`. Returns `{ access_token, token_type, expires_in, user: { id, username, role, ... } }`. Per RESEARCH.md verified endpoint.
- `refreshAccessToken(serverUrl, refreshToken)` -- POST to `{serverUrl}/api/v1/auth/refresh` with `{ refresh_token: refreshToken }`. Returns new `{ access_token, expires_in }`. Per D-79.
- `ensureValidToken()` -- reads current token from lemongridStore. If token expires within 2 minutes (D-42, D-86): try refreshAccessToken first. If refresh fails AND rememberMe AND encryptedPassword exists: decrypt password and re-login. If all fail: throw auth error. If token still valid: return it.
- `getUserProfile(serverUrl, token)` -- GET `{serverUrl}/api/v1/auth/me`. Returns user profile with role, quota. Per D-91.
- `encryptPassword(password)` / `decryptPassword(encrypted)` -- AES-GCM encryption via Web Crypto API with device-derived key (per D-78, D-77). Use `crypto.subtle.importKey` with a static salt + deriveKey from a device identifier. Store as base64 string.
- `syncAuthToBridge()` -- after login/refresh, call `sendBridgeMessage('settings.set', { key: 'lemongrid', value: { accessToken, refreshToken, serverUrl } })` so the main.js Bridge handlers can inject the JWT. This is called automatically by loginToLemonGrid and refreshAccessToken after success.

Mode change handler (per D-48, D-46, D-75):
```typescript
const handleModeChange = async (mode: 'direct' | 'cluster') => {
  if (mode === connectionMode) return;
  // D-48: Block mode switching while LemonGrid tasks are running
  if (mode === 'direct') {
    const tasks = useLemonGridStore.getState().tasks;
    const runningTasks = Object.values(tasks).filter(t =>
      ['PENDING', 'QUEUED', 'SYNCING', 'RUNNING'].includes(t.status));
    if (runningTasks.length > 0) {
      alert('有正在运行的集群任务，请等待完成后再切换模式');
      return;
    }
  }
  setConnectionMode(mode);
  if (mode === 'cluster') {
    // D-75: Auto-open login modal if not authenticated
    const lg = useLemonGridStore.getState();
    if (!lg.isConnected || !lg.accessToken) {
      setShowLoginModal(true);
    } else {
      // D-73: Try silent token validation
      try {
        await ensureValidToken();
      } catch {
        setShowLoginModal(true);
      }
    }
  }
};
```

| T-06-03 | I | localStorage token storage | mitigate | Token expires in 30min, no PII in token, HTTPS only in production |
