## Interaction States

### Preset Toolbar (new component at top of dynamic-form section)

**Location:** Inside the "参数设置" control section in `Draw.tsx`, positioned above the form fields area. Per D-04, it sits at the top of the workflow parameter area.

**Layout:** Horizontal toolbar row containing:
1. **Preset dropdown selector** (left, takes remaining width) -- native `<select>` styled to match existing `.workflow-dropdown select` pattern
2. **Add button** (+) -- small icon button, 28x28px
3. **Settings/edit button** (gear symbol) -- small icon button, 28x28px
4. **Import button** -- small icon button, 28x28px
5. **Export button** -- small icon button, 28x28px

**States:**
- No workflow selected: entire toolbar disabled, opacity 0.5
- No presets: dropdown shows "无预设" placeholder, disabled
- Loading: dropdown shows "加载中...", all buttons disabled
- Preset selected: dropdown shows preset name, edit/delete available

### Dropdown Selector
- Follows `.workflow-dropdown select` styling exactly (12px/14px padding, 8px border-radius, same background/border/color)
- Options sorted by `updatedAt` descending (most recently modified first, per RESEARCH.md Claude's Discretion)
- Currently applied preset has `[*]` prefix or bold weight

### Add Button (+)
- Creates new preset with auto-name "预设 N" (per D-06)
- Saves current `inputValues` + `imageFilenames` immediately
- After save, new preset becomes selected in dropdown

### Settings/Edit Button
- Opens inline rename/delete controls for the currently selected preset
- Rename: text input replaces dropdown text, Enter to confirm, Escape to cancel
- Delete: triggers destructive confirmation dialog (per D-07)

### Import/Export Buttons
- Export: saves currently selected preset JSON to file via Bridge
- Import: triggers file selection via Bridge, then applies D-14 conflict resolution

### Confirmation Dialogs (Custom Modal)

**Delete confirmation** (per D-07):
- Overlay: `rgba(2,6,23,0.7)` (matches `.workflow-toolkit-overlay`)
- Modal: `#0b1220` background, 14px border-radius (matches `.workflow-toolkit-modal`)
- Header: "删除预设" (15px, weight 600)
- Body: "确定要删除预设「{name}」吗？此操作不可撤销。"
- Actions: "取消" (secondary) + "删除" (destructive, `#f56565` background)

**Unsaved changes confirmation** (per D-08):
- Same modal pattern
- Header: "未保存的修改"
- Body: "当前参数有未保存的修改，切换预设将丢失这些修改。是否继续？"
- Actions: "取消" (secondary) + "不保存并切换" (accent blue)

**Import conflict** (per D-14):
- Same modal pattern
- Header: "预设名称冲突"
- Body: "已存在同名预设「{name}」，请选择处理方式。"
- Actions: "覆盖" (destructive) + "跳过" (secondary) + "重命名" (accent blue)

### Image Reference Invalidation (per D-11)
When a preset is applied and an image reference is invalid:
- The image input field shows a warning banner below the upload area
- Warning text: "图片引用已失效，请重新上传" (12px, `#f56565` color)
- Warning has a subtle red-tinted background: `rgba(245,101,101,0.1)` with dashed red border
