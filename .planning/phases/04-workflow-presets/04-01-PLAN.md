## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebApp -> Bridge | Preset file operations cross from webapp to UXP native filesystem |
| Bridge -> Filesystem | UXP handler writes/reads files in data folder |
| Import flow | User-provided JSON files enter the system via native file picker |
| Export flow | Preset data written to user-chosen location via native save dialog |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | preset filenames | mitigate | Use `sanitizeFilename()` on all filenames before filesystem operations -- prevents path traversal |
| T-04-02 | Tampering | imported preset JSON | mitigate | `validatePresetData()` performs strict schema validation -- rejects malformed JSON |
| T-04-03 | Information Disclosure | preset file content | accept | Preset files contain only parameter values, no secrets or PII. Stored in plugin data folder with OS-level access control. |
| T-04-04 | Denial of Service | preset.list handler | accept | Filesystem operations are local only; concurrent read impact is minimal in single-user plugin context. |
| T-04-05 | Tampering | preset.import file picker | mitigate | Native OS file picker restricts selection to user-accessible files only. `validatePresetData()` validates content before use. |
| T-04-06 | Tampering | preset.export save dialog | accept | Native OS save dialog writes to user-chosen location. No sensitive data in presets beyond parameter values. |
</threat_model>
