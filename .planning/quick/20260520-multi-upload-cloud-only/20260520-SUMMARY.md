# Quick Task Summary: Restrict multi-image upload to cloud model workflows

## What changed

Modified `Draw.tsx` to restrict multi-image upload to `THIRD_PARTY_API` (云端模型) templates only. ComfyUI workflow templates (`template_type === 'COMFYUI'`) now show single-image upload that replaces the existing image on re-upload.

### Changes

1. **`handleTemplateImageUpload`** — Added `replace` parameter. When `replace=true`, the upload replaces the existing image instead of appending to the array. ComfyUI templates pass `replace=true`, cloud model templates pass `replace=false`.

2. **`case 'image':` UI rendering** — Added `isCloudModel` check based on `selectedTemplate?.template_type`:
   - **Cloud model (THIRD_PARTY_API):** Multi-image UI with "添加图片" button, allows appending
   - **ComfyUI workflow:** Single-image UI — shows "上传图片" / "替换图片", hides add button when an image already exists, replaces on re-upload
