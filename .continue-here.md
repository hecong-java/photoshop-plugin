This session focused on fixing bugs and implementing small improvements to the Photoshop ComfyUI plugin.

## Work Completed

### 1. Layer/Selection Export Size Fix
- **Problem:** Images exported from PS layers/selections were using full canvas size instead of actual layer content size
- **Solution:** Implemented bounds-based crop approach:
  - Get layer bounds via batchPlay
  - Crop to bounds using DOM API `document.crop({ left, top, right, bottom })`
  - Skip crop when layer fills entire canvas
  - Handle path separator normalization (both `/` and `\`)
- **Files:** `PS-plugin/ningleai/main.js`
- **Commit:** `cbc5335`

### 2. History Rerun/Re-edit Workflow Matching Fix
- **Problem:** Re-run and re-edit from history were matching wrong workflow and not restoring parameters correctly
- **Solution:** Implemented workflow name matching:
  - Prioritize exact `workflow_name` match
  - Fall back to partial match, then scoring
  - Normalize path separators
  - Extract params from nested history structure
- **Files:** `code/webapp/src/pages/Draw.tsx`, `code/webapp/src/stores/historyStore.ts`
- **Commit:** `1363a35`

### 3. History Error Entry Filtering
- **Problem:** Error status entries were showing in history when user wanted only successful ones
- **Solution:** Filter out entries where `status_str === 'error'` (checking both direct and nested in `status` object)
- **Files:** `code/webapp/src/stores/historyStore.ts`
- **Commit:** `3c85916`
