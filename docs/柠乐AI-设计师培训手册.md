## 1. 产品简介

### 1.1 什么是柠乐AI？

**柠乐AI** 是一款集成在 Photoshop 中的 AI 图像生成插件，通过与 ComfyUI（Stable Diffusion 图形化工作流工具）连接，让设计师能够在 Photoshop 中直接使用 AI 生成图像、编辑图片、批量处理等高级功能。

### 1.2 主要优势

| 优势       | 说明                  |
| -------- | ------------------- |
| **无缝集成** | 直接在 PS 面板中操作，无需切换软件 |
| **图层交互** | 一键导出/导入图层，保留透明通道    |
| **选区支持** | 支持将选区内容发送给 AI 处理    |
| **实时预览** | 生成进度实时显示，随时掌握状态     |
| **历史记录** | 完整记录生成历史，支持一键重跑     |
| **多工作流** | 支持多种 AI 模型和工作流切换    |

### 1.3 适用场景

- 🎨 **创意灵感**：快速生成概念图、草图变体
- ✏️ **图像编辑**：局部修改、风格转换、背景替换
- 🖼️ **素材生成**：批量生成设计素材、纹理、背景
- 📱 **UI设计**：图标生成、界面元素设计
- 🎭 **风格迁移**：将照片转换为特定艺术风格

## 2. 安装与配置

### 2.1 系统要求

| 项目           | 要求                                |
| ------------ | --------------------------------- |
| Photoshop 版本 | Adobe Photoshop 2024 (24.3) 或更高版本 |
| 操作系统         | Windows 10/11 或 macOS 10.15+      |
| 网络环境         | 需要连接到运行 ComfyUI 的服务器              |
| 内存建议         | 16GB 以上                           |

### 2.2 安装步骤

#### 方式一：双击安装（推荐）

1. 双击 `.ccx` 插件文件
2. 等待 Adobe Creative Cloud 自动安装
3. 重启 Photoshop

#### 方式二：批处理安装

1. 双击 `install.bat` 文件
2. 等待安装完成
3. 重启 Photoshop

#### 方式三：手动安装

1. 将 `ningleai` 文件夹复制到以下目录：
   - **Windows**: `C:\Program Files\Adobe\Adobe Photoshop 2024\Plug-ins\`
   - **macOS**: `/Applications/Adobe Photoshop 2024/Plug-ins/`
2. 重启 Photoshop

### 2.3 首次配置

#### 步骤一：打开插件面板

1. 启动 Photoshop
2. 在菜单栏选择 **增效工具** → **柠乐AI**
3. 插件面板将在右侧显示

1. 点击顶部导航的 **「设置」** 标签
2. 在「ComfyUI 服务器地址」输入框中填写服务器地址

3. 点击 **「测试连接」** 按钮
4. 当所有能力项显示绿色 ✓ 时，表示连接成功

插件提供三大类工作流，覆盖图像处理的各种需求：

**📂 生成与风格**

| --------------- | ------------- |
| Nano Banana 2   | AI 图像变体生成     |
| Nano Banana Pro | 高级图像变体（效果更精细） |
| Seedream 4.5    | 智能图像编辑        |
| 扩图              | 图像边缘扩展        |

**📂 增强与编辑**

| 工作流  | 功能          |

| 光影重绘 | 调整产品光影      |
| 4K超清 | 超分辨率放大到 4K  |
| 2K超清 | 超分辨率放大到 2K  |

**📂 操作与变换**

| 工作流  | 功能       |
| ---- | -------- |
| 单图编辑 | 单张图片编辑处理 |
| 双图编辑 | 双图合成编辑   |
| 移除物品 | 清理图中杂物   |
| 洗图   | 图片清洗优化   |

> 💡 详细使用方法请参阅 [第5章 工作流操作指南](#5-工作流操作指南)

#### 如何选择工作流

1. 点击工作流下拉菜单
2. 根据您的需求选择对应的工作流
3. 页面将自动加载该工作流的参数表单

### 4.2 参数配置

#### 文本参数

| 参数名       | 说明       | 使用技巧                              |
| --------- | -------- | --------------------------------- |
| **正向提示词** | 描述您想要的内容 | 越具体越好，可使用英文或中文                    |
| **负向提示词** | 描述不想要的元素 | 常用：blurry, low quality, watermark |

#### 数值参数

| 参数名       | 范围       | 说明                  |
| --------- | -------- | ------------------- |
| **采样步数**  | 1-50     | 步数越多细节越丰富，但生成更慢     |
| **CFG强度** | 1-20     | 值越高越遵循提示词，太大会导致图像僵硬 |
| **生成宽度**  | 512-2048 | 输出图片宽度（建议64的倍数）     |
| **生成高度**  | 512-2048 | 输出图片高度（建议64的倍数）     |
| **随机种子**  | 任意数字     | 相同种子+相同参数=相同结果      |

#### 下拉选择参数

| 参数名     | 说明                |
| ------- | ----------------- |
| **采样器** | 不同的采样算法，影响生成速度和质量 |
| **模型**  | 不同的AI模型有不同的风格特点   |
| **调度器** | 控制采样过程的调度方式       |

### 4.3 图片上传

#### 三种上传方式

**1. 从本地上传**

- 点击「从本地上传」按钮
- 选择本地图片文件
- 支持 PNG、JPG、WEBP 格式

**2. 从 PS 图层导入**

- 在 Photoshop 中选中要导出的图层
- 点击「从 PS 图层加载」按钮
- 图层内容将自动导入到插件中

**3. 从 PS 选区导入**

- 使用选区工具选择要处理的区域
- 点击「从 PS 选区加载」按钮
- 选区内容将被导出（包含透明通道）

#### 图片要求

| 项目  | 建议         |
| --- | ---------- |
| 格式  | PNG（保留透明度） |
| 尺寸  | 与目标生成尺寸相近  |
| 分辨率 | 72-300 dpi |

### 4.4 生成与监控

#### 开始生成

1. 确认所有必要参数已填写
2. 点击「开始生成」按钮
3. 系统将任务提交到服务器队列

#### 进度监控

生成过程中，您将看到：

```
┌─────────────────────────────────────┐

│ ████████████░░░░░░░░░░░░░░░░░░░░░░░ │
```

#### 队列状态

| 状态         | 说明         |

### 4.5 结果处理

生成完成后，您可以：

## 5. 工作流操作指南

本章详细介绍每个工作流的功能和使用方法，按三大类别组织。

### 5.1 📂 生成与风格

#### Nano Banana 2 / Pro - AI 图像变体

**功能**：根据参考图生成多角度、多姿势的变体图像

**适用场景**：

- 人物多角度变体
- 产品多角度展示
- 创意灵感生成

**操作步骤**：

1. 上传一张参考图片（人物、产品等）
2. 填写提示词描述变化方向
   ```
   示例：create a 2x2 grid of this woman,
   various different poses and facial expressions,
   maintain her exact facial features,
   no borders, labels
   ```
3. 选择宽高比（1:1、9:16、16:9 等）
4. 选择分辨率（2K 等）
5. 点击生成

***

#### Seedream 4.5 - 智能图像编辑

**功能**：风格转换、细节调整、画面重组

**适用场景**：

- 风格迁移
- 细节优化
- 画面调整

**操作步骤**：

1. 上传需要编辑的图片
2. 填写提示词描述编辑效果
3. 选择宽高比和质量级别
4. 点击生成

***

#### 扩图 - 图像边缘扩展

**功能**：延伸画面边界、补全缺失区域

**适用场景**：

- 扩展图片边界
- 补全缺失内容
- 调整图片比例

**操作步骤**：

1. 上传需要扩展的图片
2. 设置扩展参数
   - 上/下/左/右：各方向扩展像素数
   - 羽化：边缘过渡平滑度（建议 50-100）
3. 填写提示词描述扩展区域内容
   ```
   示例：continue the scene naturally, extend the background
   ```
4. 点击生成

***

### 5.2 📂 增强与编辑

#### 抠图 - 智能去除背景

**功能**：自动识别并抠出物体，生成透明背景

**适用场景**：

- 抠产品
- 抠人物

## 6. 常见问题解答

### Q1: 无法连接到服务器怎么办？

**检查清单**：

1. 确认服务器地址是否正确
2. 检查网络连接是否正常
3. 确认 ComfyUI 服务是否正在运行
4. 联系 IT 管理员确认服务器状态

### Q2: 生成速度很慢怎么办？

**可能原因与解决方案**：

- 服务器负载高：等待队列消化，或选择低分辨率
- 参数步数太高：降低采样步数（20-25）
- 模型较大：选择更轻量的模型

### Q3: 生成结果不满意怎么办？

**优化建议**：

1. **提示词不够具体**：添加更多描述细节
2. **CFG 太高/太低**：调整到 7-10 范围
3. **负向提示词缺失**：添加不想要的元素
4. **模型不适合**：尝试其他模型

### Q4: 导入到 PS 后图片位置不对？

**解决方案**：

1. 导入前先确认文档尺寸
2. 使用移动工具调整位置
3. 考虑先生成再手动放置

### Q5: 从 PS 图层导入失败？

**检查**：

1. 确认已选中图层
2. 确认图层可见（眼睛图标打开）
3. 确认图层有内容（不是空白图层）

### Q6: 选区导入没有效果？

**检查**：

1. 确认已创建选区（有蚂蚁线）
2. 确认选区在当前文档上
3. 尝试重新创建选区

### Q7: 如何保存我的常用设置？

设置会自动保存在浏览器本地存储中，下次打开时会自动恢复。

### Q8: 生成过程中可以取消吗？

当前版本提交后无法取消，但可以在服务器管理端清除队列。

## 9. 附录：高级配置

### 9.1 node-config.json 配置说明

`node-config.json` 是一个重要的配置文件，用于控制 ComfyUI 节点的哪些参数会显示在 Photoshop 插件的 UI 界面中。

#### 配置文件位置

```
PS-plugin/ningleai/node-config.json
```

#### 配置文件结构

```json
{
  "$comment": "配置说明...",
  "version": "1.0",
  "nodes": [
    {
      "class_type": "节点类型名称",
      "inputs": ["参数1", "参数2"]
    }
  ]
}
```

#### 字段说明

| 字段          | 类型       | 说明                                          |
| ------------- | ---------- | --------------------------------------------- |
| `version`     | 字符串     | 配置文件版本号                                |
| `nodes`       | 数组       | 节点配置列表                                  |
| `class_type`  | 字符串     | ComfyUI 节点的类型名称（**区分大小写**）       |
| `inputs`      | 数组（可选）| 要显示的参数名称列表                          |

#### 配置规则

1. **指定 inputs 数组**：只显示数组中列出的参数
   ```json
   {
     "class_type": "LoadImage",
     "inputs": ["image"]
   }
   ```
   → LoadImage 节点只显示 `image` 参数

2. **省略 inputs 或留空数组**：显示该节点的所有参数
   ```json
   {
     "class_type": "CLIPTextEncode"
   }
   ```
   → CLIPTextEncode 节点显示所有可用参数

3. **节点未在配置中列出**：使用工作流默认值，不在 UI 中显示任何参数

#### 配置示例

**完整配置示例**：

```json
{
  "$comment": "ComfyUI Node Display Configuration",
  "version": "1.0",
  "nodes": [
    {
      "class_type": "LoadImage",
      "inputs": ["image"]
    },
    {
      "class_type": "CLIPTextEncode"
    },
    {
      "class_type": "KIE_NanoBanana2_Image"
    },
    {
      "class_type": "KIE_NanoBananaPro_Image"
    },
    {
      "class_type": "KIE_Seedream45_Edit"
    }
  ]
}
```

**配置效果说明**：

| 节点类型                    | UI 显示行为                     |
| --------------------------- | ------------------------------- |
| LoadImage                   | 只显示图片选择参数              |
| CLIPTextEncode              | 显示所有参数（正向/负向提示词） |
| KIE_NanoBanana2_Image       | 显示所有 Nano Banana 2 参数     |
| KIE_NanoBananaPro_Image     | 显示所有 Nano Banana Pro 参数   |
| KIE_Seedream45_Edit         | 显示所有 Seedream 4.5 参数      |
| 其他节点                    | 不显示，使用工作流默认值        |

### 9.2 如何添加新节点配置

#### 步骤一：获取节点类型名称

1. 在 ComfyUI 中打开对应的工作流 JSON 文件
2. 找到目标节点，查看其 `type` 字段值
   ```json
   {
     "id": 1,
     "type": "KSampler",  // 这就是 class_type
     ...
   }
   ```

#### 步骤二：确定要显示的参数

1. 在工作流 JSON 中找到该节点的 `inputs` 部分
   ```json
   {
     "type": "KSampler",
     "inputs": [
       {"name": "steps"},
       {"name": "cfg"},
       {"name": "sampler_name"},
       {"name": "seed"}
     ]
   }
   ```

#### 步骤三：添加到配置文件

```json
{
  "class_type": "KSampler",
  "inputs": ["steps", "cfg", "sampler_name"]
}
```

> 💡 注意：`seed` 参数通常不需要用户手动输入，可以不包含在 inputs 中

### 9.3 常见问题

#### Q1: 修改配置后没有生效？

- 确认 JSON 格式正确（可用在线 JSON 验证器检查）
- 重启 Photoshop 或重新加载插件
- 清除浏览器缓存（插件使用 WebView）

#### Q2: 参数显示不正确？

- 检查 `class_type` 是否与工作流中的节点类型完全一致（区分大小写）
- 确认 `inputs` 数组中的参数名称拼写正确

#### Q3: 如何隐藏某个参数？

- 在 `inputs` 数组中不包含该参数名即可
- 或者完全不配置该节点，使用工作流默认值

### 9.4 配置最佳实践

| 建议                       | 说明                                     |
| -------------------------- | ---------------------------------------- |
| **精简显示参数**           | 只暴露用户需要调整的参数，减少界面复杂度 |
| **保留默认值**             | 技术性参数（如 seed）可使用工作流默认值  |
| **区分大小写**             | class_type 必须与节点类型完全匹配        |
| **测试验证**               | 修改后务必测试工作流是否正常运行         |
| **版本管理**               | 建议保留配置文件的备份                   |
